Deterministic execution firewall for AI agents.

Sentinel sits between autonomous AI agents and real infrastructure,
verifying identity, enforcing deterministic policy decisions,
and controlling how actions are executed.

## Security Model

Every action must pass multiple security checks before execution.

### Agent Identity
Agents authenticate using **Ed25519 cryptographic signatures**.

### Signed Requests
Requests are signed and timestamped to prevent tampering or replay.

### Deterministic Policy Engine
Actions are evaluated using strict policy rules:

ALLOW  
REVIEW  
DENY  

### Controlled Execution
Approved actions execute only through safe adapters.

### Tamper-Evident Telemetry
All actions are recorded in a cryptographic audit chain.

---

## Threat Model

Sentinel protects infrastructure from unsafe or unauthorized actions
generated by autonomous systems.

Threats mitigated:

• unauthorized agent execution  
• prompt-injected agent commands  
• replay attacks  
• unsafe automation loops  
• compromised agents

## Use Cases

Sentinel is designed for environments where autonomous systems interact with infrastructure.

### AI DevOps Agents
Control infrastructure automation safely.

### Autonomous Systems
Ensure AI systems cannot directly execute unsafe commands.

### Multi-Agent Environments
Enforce global policy across multiple agents.

Example:

Agent → Sentinel → Infrastructure
