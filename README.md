Deterministic execution firewall for AI agents.

Sentinel is an AI agent security gateway that sits between autonomous agents and real infrastructure.

It verifies agent identity, validates signed action requests, applies policy decisions, records tamper-evident audit logs, and routes approved actions through a controlled execution layer.

---

## Security Model

Sentinel enforces deterministic authorization between AI agents and real infrastructure.

Every action must pass these checks before execution:

1. **Agent Identity**
   - Ed25519 cryptographic identity
   - Registered agent registry
   - Revocation / suspension support

2. **Signed Action Requests**
   - Canonical command digest
   - Ed25519 signature verification
   - Timestamp validation
   - Replay protection

3. **Deterministic Policy Engine**
   - allow / review / deny decisions
   - validator hard-lock policy
   - reputation-aware enforcement

4. **Controlled Execution**
   - Executor worker queue
   - Safe adapter layer
   - Infrastructure interaction

5. **Tamper-Evident Telemetry**
   - audit chain
   - decision trace
   - operator visibility

---

## Threat Model

Sentinel protects infrastructure from unsafe or unauthorized actions generated by autonomous systems.

### Threats Sentinel Mitigates

**Unauthorized agent execution**
- Agents must possess a valid Ed25519 identity.
- All requests must be cryptographically signed.

**Prompt-injected or manipulated agent actions**
- All actions pass deterministic policy evaluation.
- LLM outputs cannot directly trigger execution.

**Replay attacks**
- Timestamp validation
- Nonce / replay protection

**Unsafe automation**
- High-risk commands are automatically denied.
- Sensitive operations require human approval.

**Agent compromise**
- Agents can be suspended or revoked through the identity registry.

## Use Cases

Sentinel is designed for environments where autonomous systems interact with real infrastructure.

### AI Agents with Infrastructure Access
Protect systems where LLM agents can:
- restart services
- call APIs
- trigger automation
- modify infrastructure

Sentinel ensures every action is verified and policy-approved before execution.

### DevOps Automation
Insert Sentinel between automation tools and infrastructure to enforce deterministic execution policy.

Example:
Agent → Sentinel → Infrastructure

This prevents unsafe automation scripts or agent mistakes from executing directly.

### Validator Infrastructure (Web3)

Sentinel Validator Edition provides deterministic enforcement for validator automation.

Example protections:

- unauthorized service restarts
- unsafe remediation scripts
- automation escalation loops
- replayed automation commands

### Multi-Agent Systems

In environments with multiple autonomous agents, Sentinel ensures:

- each agent has a cryptographic identity
- actions are verifiable
- execution policy is enforced globally

### Out of Scope

Sentinel does not protect against:

- Compromise of the host operating system
- Root-level attacker control of the Sentinel server
- Physical hardware attacks

# What Sentinel Does

Sentinel provides:

• Ed25519 agent identity and signed requests  
• Timestamp and replay protection  
• Strict command validation  
• Policy-based allow / review / deny decisions  
• Reputation-aware decision support  
• Human approval workflow for sensitive actions  
• Adapter-based executor control  
• Append-only tamper-evident audit chain  
• Telemetry and operator dashboard  
• Python SDK for agent integration
